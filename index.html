Local & Private — Your files never leave your device

Client-Side Processing (CDN scripts loaded on init)

Privacy Guarantee

  • Your files never leave your device. All sanitization runs locally inside your browser using WebAssembly (Pyodide).
  • No file contents are uploaded, transmitted, or stored on any server.
  • No analytics, tracking, cookies, or telemetry of any kind.
  • Works fully offline after the initial library load.
  • You can disconnect from the internet after the engine loads and it will still work.
  • Prefer maximum privacy? Right-click this page, choose "Save As," and save the HTML file to your computer. Then open it directly from your desktop — no server or internet hosting needed. The entire application is this single file.

What It Detects

  • Personal Info: Names (contextual detection), email addresses, phone/fax numbers
  • Identifiers: Social Security Numbers, EINs, credit card numbers, account/reference numbers
  • Location Data: Street addresses, city/state/ZIP, PO Boxes
  • Financial: Dollar amounts, compensation figures
  • Other: Dates, URLs, IP addresses, company/organization names
  • Each category can be toggled on/off independently before processing.

Known Limitations

  • Best-effort redaction. This tool uses pattern matching (regex), not AI. It may miss unconventional formats or context-dependent data.
  • Name detection is imperfect. Uncommon names, names in running text without contextual clues, or non-English names may not be caught.
  • DOCX run-splitting: Microsoft Word sometimes splits a single word across internal XML elements. A name or date split across runs may not be detected.
  • PDF output is simplified. Sanitized PDFs are re-created as plain text — original formatting, images, and layout are not preserved.
  • Max file size: 10 MB per file. Larger files may cause browser memory issues.
  • Supported formats: XLSX, DOCX, PDF, TXT, CSV only.
  • Always manually review sanitized output before sharing externally.

Security Considerations

  • CDN dependency: On first load, JavaScript libraries (Pyodide, JSZip) and Python packages are fetched from CDNs. These requests are code-only — no file data is sent.
  • Integrity checks: CDN scripts use Subresource Integrity (SRI) hashes to prevent tampering. Python packages are pinned to exact versions.
  • No server component: There is no backend, no API, no database. The HTML file is the entire application.
  • Not a substitute for professional review. For highly sensitive or regulated documents (HIPAA, SOX, GDPR), pair this tool with manual review by qualified personnel.
  • Open source & auditable. The entire source code is visible by viewing this HTML file — nothing is hidden or obfuscated.
